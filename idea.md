0. Концепция

GroBro – личный ассистент-дневник для выращивания (indoor plants & legal cannabis)

Функции:

Дневник и трекинг всего цикла.

Расчёт поливов / напоминания.

Фото-диагностика состояния.

“Дебаг-консоль” — разговорный помощник, который помогает понять, что могло пойти не так.

База знаний и подсказки “что дальше”.

1. Основные сценарии пользователя

Я хочу завести журнал своего гроубокса / палатки.

Добавить растение, сорт (опционально), дату старта.

Писать заметки по дням: поливы, изменения, стресс, пересадка.

Видеть таймлайн роста.

Я хочу не забывать, когда поливать / подкармливать.

Я задаю тип субстрата, объём горшка, тип воды.

GroBro предлагает базовый интервал полива + даёт напоминания.

Я отмечаю: “полил”, “сделал промывку”, “дал удобрение”.

Я хочу понять, что не так с растением.

Делаю фото.

Приложение примерным образом определяет: “перелив / недолив / ожог светом / возможный дефицит / вредители” — максимально аккуратно, без жёстких диагнозов.

Даёт базовые безопасные советы уровня “проверь влажность субстрата”, “убери лампу дальше”, “осмотри нижнюю сторону листьев”.

Я хочу видеть всю историю и статистику.

Сколько дней прошло с начала цикла.

Сколько было поливов, каких объёмов.

График веса горшка (если пользователь фиксирует), заметки, тэги “стресс / обрезка / пересадка”.

2. Структура приложения
2.1. Домашний экран “Garden”

Карточки растений:

Фото (последний снимок).

Имя / сорт.

Стадия (seedling / veg / flower или просто “Stage 1/2/3”).

Индикатор: “полив через X дней” / “полив просрочен на X дней”.

Быстрые действия:

Полив

Фото-осмотр

Заметка

2.2. Профиль растения

Вкладки:

Overview

Фото-обложка.

Основные параметры:

Дата посадки / возраст.

Тип субстрата (коко, земля, гидро и т.д. — без углубления в рецепты).

Объём горшка.

Освещение: число часов света / тип лампы (без конкретных “рецептов выращивания”).

Статус: “Всё ок / Требует внимания / Недавний стресс”.

Diary (дневник)

Лента событий: поливы, подкормки, пересадки, обрезка, заметки, фото.

Фильтры по типу событий.

Watering

Текущий расчёт “рекомендуемый интервал”.

История поливов с объёмом.

Возможность отмечать: “слишком влажно / слишком сухо” — для последующей авто-коррекции.

Debug

История “проблемных” фото и комментариев.

Кнопка “разобраться”: запускает AI-диагностику на базе последнего фото + дневника.

3. Модуль “Полив & напоминания”

Без конкретных “рецептов”, но с умной логикой:

При создании растения пользователь указывает:

Тип субстрата.

Объём горшка.

Примерную температуру / влажность (диапазоны).

Приложение:

Вычисляет стартовый интервал для напоминаний (например, 3–5 дней).

Позволяет пользователю руками “подтюнить” (сдвинуть в меньшую/бОльшую сторону).

После каждого полива:

Пользователь может указать “слишком рано / в самый раз / поздно”.

Модель подстраивает интервал под конкретное растение.

Идея:
Ты не учишь пользователя, как выращивать, а помогаешь не забывать и отслеживать персонализированный режим.

4. Фото-диагностика (Vision + Apple Intelligence)
4.1. Входные данные

Фото растения (желательно нескольких ракурсов).

Контекст:

Возраст.

Последние поливы / пересадки.

Замеченные пользователем вещи (“листья пожелтели снизу / пятна на верхних листьях”).

4.2. L1: Локальная модель (Core ML / Vision)

Локальный классификатор:

Классы очень общие и безопасные:

Hydration: норм / возможный перелив / возможный недолив.

Light stress: нет / возможный.

Leaf condition: норм / общая хлорозность / пятна / некроз.

Pests suspicion: да / нет (без названия конкретных вредителей).

Плюс просто флаг “подозрительно” vs “всё ок”.

4.3. L2: Языковая модель (Apple Foundation Model или облако)

Языковая модель не даёт незаконных инструкций, а:

Переформулирует результат в понятный язык.

Даёт мягкие, общие советы:

“Проверь, не остаётся ли вода в поддоне”.

“Посмотри, не слишком ли близко лампа”.

“Если сомневаешься — сравни с референс-фото здоровых листьев того же вида”.

Может предложить чек-лист:

1–2 пункта, что проверить.

1–2 пункта, как наблюдать дальше (не “сделай X грамм удобрения Y”, а “на ближайшие N дней наблюдай и при необходимости обратись к общим гайдлайнам”).

5. Debug Console (“Что могло пойти не так?”)

Отдельный экран-чат для каждого растения:

Слева: контекст (возраст, последние поливы, события).

Справа: диалог с ассистентом.

Функциональность:

Пользователь пишет:
“Листья снизу пожёлтели, поливал 2 дня назад, лампа 30 см”.

Модель:

Смотрит историю дневника (но не лезет в рецепты).

Предлагает гипотезы без конкретных рецептур:

“Это может быть возрастное пожелтение старых листьев или стресс. Попробуй: проверить влажность субстрата пальцем на глубине; убедиться, что нет прямого перегрева; сравнить новые листья с прошлыми фото”.

Дальше пользователь делает пометки, что помогло / не помогло — и это идёт в историю “Debug”.

6. База знаний (очень аккуратная)

Раздел “Guides”:

Статьи максимально общего уровня:

Как вести дневник растения.

Как проверять влажность субстрата.

Как безопасно пользоваться лампами (общие рекомендации по расстоянию, без “секретных схем”).

Как распознавать базовые признаки переувлажнения vs пересушивания.

Раздел про безопасность и законность:

Напоминание, что пользователь обязан соблюдать местные законы.

Советы по безопасности данных (ничего не публикуется, всё локально, если так).

7. AI / Apple Intelligence интеграция (по уровням)

On-device Vision / Core ML

Фото → классификатор → тех. лейблы.

On-device SystemLanguageModel (где доступен)

На основе лейблов + контекста дневника → текстовый совет.

Фолбэк в облако

Если устройство не поддерживает Apple Intelligence, можно использовать свой backend + любую LLM.

App Intents / Siri / Shortcuts

“Hey Siri, ask GroBro, когда последний раз я поливал растения”.

“Log watering for GroBro”.

8. Монетизация / уровни

Free:

До 3 растений.

Дневник, напоминания, базовая фото-диагностика раз в X дней.

Pro:

Неограниченное количество растений.

Расширенная аналитика (графики, экспорт CSV/JSON).

Частая фото-диагностика.

iCloud-синхронизация.

+ потенциал B2B / продвинутый режим (но позже):

Интеграции с датчиками (влажность/температура).

API для магазинов/гроушопов в легальных регионах.

9. Визуальный стиль / UX

Атмосфера: спокойный, “зеленый”, без “дарк” нарко-вайба.

Главная метафора: “умный дневник твоего зелёного друга”, а не “сделай супер-урожай”.

Много таймлайнов, фото-коллажей “как растение менялось”.
